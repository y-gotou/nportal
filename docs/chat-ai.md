# 会議チャットの AI アシスタント

- 作成日: 2026-07-08

## 概要

会議チャットで本文に `@AI` を含むテキストを投稿すると、社内 LLM(OpenAI 互換 API)が生成した返信が「AI アシスタント」のメッセージとして同じルームに投稿されます。LLM への接続は既存のプロキシ基盤([docs/llm-proxy.md](llm-proxy.md))を利用します。

## 動作

1. `POST /api/chat/:scheduleId/messages` で本文に `@AI` メンションを含むテキスト投稿を検知します(判定は `hasChatAiMention`。行頭・空白・記号の直後の `@AI` のみ反応し、メールアドレス等には反応しません)。
2. 投稿自体は通常どおり保存・応答し、`waitUntil` によるバックグラウンド処理で LLM を呼び出します。
3. 生成した返信を投稿者 `ai-assistant@nportal.local`(表示名「AI アシスタント」)のメッセージとして挿入し、`reply_to_id` で元投稿に紐付けます。ルームのバージョンが上がるため、クライアントには次回ポーリング(3 秒間隔)で表示されます。
4. 生成に失敗した場合は、その旨のメッセージを AI アシスタントとして投稿します(ユーザーを無応答で待たせない)。

## LLM への入力

- そのルームの直近 20 件のテキストメッセージ(スタンプ・削除済み・空本文は除外)を表示名付きで整形して渡します。
- システムプロンプトに勉強会のタイトルと開催日を含めます。
- 画像・ファイル添付は入力に含めません。
- 応答は `max_tokens: 1024` で生成し、チャットの最大本文長(2000 文字)に切り詰めます。

## 使用モデル

環境変数 `LLM_CHAT_MODEL` に設定したモデル ID を使用します。未設定の場合は上流の `/v1/models` の先頭のモデルを使用します。

## 実装

| ファイル | 役割 |
| --- | --- |
| `utils/chat.ts` | `CHAT_AI_EMAIL` / `CHAT_AI_DISPLAY_NAME` / `hasChatAiMention`(クライアント・サーバー共有) |
| `server/utils/chat-ai.ts` | コンテキスト整形・LLM 呼び出し・返信投稿(`postChatAiReply`) |
| `server/api/chat/[scheduleId]/messages.post.ts` | メンション検知とバックグラウンド実行の起動 |

## 制約

- 読み取り専用ルーム(開催日の翌日以降)では投稿自体ができないため、AI 返信も発生しません。
- AI アシスタントのメッセージが `@AI` を含んでいても再帰的には反応しません(トリガーはユーザー投稿の API のみ)。
- メンション 1 回につき LLM 呼び出しは 1 回です。レート制限は設けていません。
