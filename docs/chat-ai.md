# 会議チャットの AI アシスタント

- 作成日: 2026-07-08

## 概要

会議チャットで本文に `@AI` を含むテキストを投稿すると、社内 LLM(OpenAI 互換 API)が生成した返信が「AI アシスタント」のメッセージとして同じルームに投稿されます。LLM への接続は既存のプロキシ基盤([docs/llm-proxy.md](llm-proxy.md))を利用します。

## 動作

1. 本文に `@AI` メンションを含むテキストを投稿すると(判定は `hasChatAiMention`。行頭・空白・記号の直後の `@AI` のみ反応し、メールアドレス等には反応しません)、投稿したクライアントが `POST /api/chat/:scheduleId/ai-reply` を呼び出します。
2. サーバーはリクエスト中に同期で LLM を呼び出して返信を生成します(生成中はクライアントに「AI アシスタントが返信を作成中です…」と表示。LLM の生成に数分かかることがあるため、投稿者はタブを開いたままにする必要があります)。
   - `waitUntil` によるバックグラウンド生成は採用していません。レスポンス返却後の isolate は破棄されることがあり、生成が完走しない場合があるためです。
   - 同じ投稿への AI 返信が既に存在する場合は再生成しません(`hasChatAiReplyTo` による重複ガード)。
3. 生成した返信を投稿者 `ai-assistant@nportal.local`(表示名「AI アシスタント」)のメッセージとして挿入し、`reply_to_id` で元投稿に紐付けます。ルームのバージョンが上がるため、全クライアントにポーリング(3 秒間隔)で表示されます。
4. 生成に失敗した場合は、その旨のメッセージを AI アシスタントとして投稿します(ユーザーを無応答で待たせない)。

## 入力補完

メッセージ入力欄で `@`(半角)を入力すると `@AI` の候補が表示されます(「AI」の前方一致で追従)。候補表示中は Enter / Tab / クリックで確定して `@AI ` を挿入し、Esc で閉じます(同じ `@` トークンの間は再表示しません)。IME 変換中のキー操作には反応しません。

## LLM への入力

- そのルームの直近 20 件のテキストメッセージ(スタンプ・削除済み・空本文は除外)を表示名付きで整形して渡します。
- システムプロンプトに勉強会のタイトルと開催日を含めます。
- 画像・ファイル添付は入力に含めません。
- 応答は `max_tokens: 1024` で生成し、チャットの最大本文長(2000 文字)に切り詰めます。

## Web 検索

`TAVILY_API_KEY` が設定されている場合、回答生成の前に次の 2 段階処理を行います。

1. **要否判定**: LLM への小さな呼び出し(`max_tokens: 200`)で「質問への回答に Web 検索が必要か・検索クエリは何か」を JSON で判定させます。
2. **検索と回答**: 必要と判定された場合のみ Tavily API(`api.tavily.com`)で検索し(上位 5 件、抜粋は 500 文字まで)、結果をプロンプトに加えて回答を生成します。参考にした URL は回答末尾に記載するよう指示します(チャット UI が自動でリンク化します)。

キー未設定・判定不能(JSON 解析失敗)・検索 API の失敗は、すべて「検索なしの回答」にフォールバックします。トンネル先の LLM サーバー側には変更も依存もありません(function calling 不使用。バックエンドやモデルの入れ替えに影響されません)。

## 使用モデル

環境変数 `LLM_CHAT_MODEL` に設定したモデル ID を使用します。未設定の場合は上流の `/v1/models` の先頭のモデルを使用します。

## 環境変数

| 変数 | 内容 |
| --- | --- |
| `LLM_CHAT_MODEL` | 使用するモデル ID(任意。未設定時は `/v1/models` の先頭) |
| `TAVILY_API_KEY` | Tavily の API キー(任意。未設定時は Web 検索なしで動作) |

## 実装

| ファイル | 役割 |
| --- | --- |
| `utils/chat.ts` | `CHAT_AI_EMAIL` / `CHAT_AI_DISPLAY_NAME` / `hasChatAiMention` / 入力補完判定 `findChatAiMentionCandidate`(クライアント・サーバー共有) |
| `app/components/chat/ChatComposer.vue` | `@AI` 候補ポップアップの表示・確定・キー操作 |
| `app/components/chat/ChatRoom.vue` | 投稿後の `ai-reply` 呼び出しと生成中表示 |
| `server/utils/chat-ai.ts` | コンテキスト整形・検索要否判定・Tavily 検索・LLM 呼び出し・返信投稿(`postChatAiReply`) |
| `server/api/chat/[scheduleId]/ai-reply.post.ts` | AI 返信生成エンドポイント(メンション検証・重複ガード・同期実行) |

## 制約

- 読み取り専用ルーム(開催日の翌日以降)では投稿自体ができないため、AI 返信も発生しません。
- AI アシスタントのメッセージが `@AI` を含んでいても再帰的には反応しません(トリガーはユーザー投稿の API のみ)。
- メンション 1 回につき LLM 呼び出しは 1 回です。レート制限は設けていません。
